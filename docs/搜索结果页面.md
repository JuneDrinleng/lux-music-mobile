# 搜索结果页面

## 当前实际使用的文件
当前移动端首页里，实际在用的搜索结果页是：

- `src/screens/Home/Vertical/SearchPage.tsx`

它不是旧的 `src/screens/Home/Views/Search/*` 直接挂出来的页面，而是竖屏首页里的一个全屏浮层。

## 入口链路

搜索结果页的打开链路如下：

1. `src/screens/Home/index.tsx`
   根据 `useHorizontalMode()` 选择进入横屏或竖屏首页。
2. `src/screens/Home/Vertical/SharedTopBar.tsx`
   点击顶部搜索框后，触发 `global.app_event.openVerticalSearchPage(...)`。
3. `src/screens/Home/Vertical/Main.tsx`
   监听 `openVerticalSearchPage` 事件，挂载 `SearchPage` 并控制显示 / 关闭。
4. `src/screens/Home/Vertical/SearchPage.tsx`
   负责输入、联想词、历史记录、搜索请求、结果渲染、收藏歌曲、添加到歌单等。

## 当前这页的核心职责

### 1. 顶部搜索栏

- 返回按钮
- 搜索输入框
- 清空输入按钮
- 结果态下的搜索类型切换按钮
  - 复用 `src/components/common/SegmentedIconSwitch.tsx`
  - 位置在搜索框下方
  - 左侧是结果标题，右侧是滑块按钮
  - 两个选项：
    - `music`：歌曲搜索结果
    - `songlist`：歌单搜索结果

注意：

- 搜索历史 / 搜索建议态不显示切换按钮
- 只有真正进入结果态才显示“标题 + 切换滑块”

### 2. 初始态

以下情况显示初始态，而不是结果列表：

- 正在编辑搜索框
- 还没有真正提交搜索词

初始态内容包括：

- 搜索建议
- 搜索历史
- 清空历史
- 点击建议词 / 历史词直接发起搜索

### 3. 歌曲搜索结果

歌曲结果由 `SearchPage.tsx` 自己渲染，单项包含：

- 封面
- 歌名
- 来源标签
- 歌手
- 时长
- 收藏按钮
- 添加到歌单按钮

点击歌曲会直接加入队列并播放。

### 4. 歌单搜索结果

歌单搜索结果也由 `SearchPage.tsx` 渲染，单项包含：

- 封面
- 歌单名
- 来源标签
- 作者 / 播放量 / 描述摘要
- 进入详情按钮

点击歌单结果后，现在会走新的歌单详情链路：

- `SearchPage.tsx`
  调用 `global.app_event.openPlaylistDetail({...})`
- `Main.tsx`
  保留搜索浮层，直接在它上面挂载歌单详情覆盖层
- `PlaylistTab.tsx`
  以独立详情模式展示歌单内容

也就是说，搜索结果里的歌单详情，现在和 HomeTab 点击歌单进入的是同一套新详情页。
区别是当它从搜索结果打开时，详情页会压在搜索页上面，返回后会回到原来的搜索结果状态。

## 数据流

### 搜索历史

- `src/core/search/search.ts`
  - `getSearchHistory()`
  - `addHistoryWord()`
  - `removeHistoryWord()`
  - `clearHistoryList()`

底层存储在：

- `src/utils/data.ts`

### 搜歌曲

- `src/core/search/music.ts`

职责：

- 按来源搜索在线歌曲
- `all` 模式下并发搜多个源
- 合并结果
- 写入搜索状态

### 歌曲搜索状态整理

- `src/store/search/music/state.ts`
- `src/store/search/music/action.ts`

职责：

- 缓存分页结果
- 多源合并
- 去重
- 根据关键词相似度排序

### 搜歌单

- `src/core/search/songlist.ts`

职责：

- 按来源搜索在线歌单
- `all` 模式下并发搜多个源
- 合并歌单结果
- 写入歌单搜索状态

### 歌单搜索状态整理

- `src/store/search/songlist/state.ts`
- `src/store/search/songlist/action.ts`

职责：

- 缓存歌单分页结果
- 多源合并
- 根据关键词相似度排序
